# 안드로이드 스터디 Fork 방법

### 1. 스터디를 진행하는 레포지토리 Fork 하기

현재 진행되는 스터디 레포지토리의 경로는 BCSD_Android_Study 오가니제이션의 study-android 레포지토리를 아래의 사진처럼 Fork 하면 됩니다.

<img src="/img/@android/jkh_study_fork/study_fork1.png" alt="이미지" style={{ marginBottom: "0px" }} />

### 2. 나의 레포지토리의 Fork 되었는지 확인

Fork 이후, 나의 레포지토리의 Fork 한 레포지토리가 잘 들어왔는 지 확인합니다.
<img src="/img/@android/jkh_study_fork/study_fork2.png" alt="이미지" style={{ marginBottom: "0px" }} />

### 3. Fork 된 레포지토리 재확인

레포지토리로 들어와서 Forked from BCSD-Android-Study/study-android 되었는 지 재확인합니다.

<img src="/img/@android/jkh_study_fork/study_fork3.png" alt="이미지" style={{ marginBottom: "0px" }} />

### 4. 레포지토리 Clone하기

> **저는 git bash를 통해서 git 작업을 진행하겠습니다. (GUI 쓰시는 분들은 잘 하실거라 믿습니다!)**
> 

Fork된 레포지토리를 다음과 같이 clone하여 내 로컬로 가져옵니다.

<img src="/img/@android/jkh_study_fork/study_fork4.png" alt="이미지" style={{ marginBottom: "0px" }} />

### 5. Clone 파일 확인하기

Clone 한 파일이 로컬에 잘 들어왔는 지 확인합니다.

<img src="/img/@android/jkh_study_fork/study_fork5.png" alt="이미지" style={{ marginBottom: "0px" }} />

### 6. Commit 및 Push 하기

현재 Unstaged 되었는 지 확인하기 위해서 `git status` 명령어로 빨간 상태(Unstaged) 확인을 합니다. 확인한 뒤, 해당 Unstaged 상태를 Staged 상태로 바꾸기 위해 `git add .` 명령어를 입력합니다.

<img src="/img/@android/jkh_study_fork/study_fork6.png" alt="이미지" style={{ marginBottom: "0px" }} />

그 다음으로 해당 파일이 Staged 상태로 잘 되었는 지 재확인하기 위해서 `git status` 명령어로 재확인합니다.

<img src="/img/@android/jkh_study_fork/study_fork7.png" alt="이미지" style={{ marginBottom: "0px" }} />


그 다음 Staged 된 파일을 커밋합니다. 아래의 명령어 처럼 `git commit -m “원하는 메시지”` 명령어를 입력합니다. 

<img src="/img/@android/jkh_study_fork/study_fork8.png" alt="이미지" style={{ marginBottom: "0px" }} />


커밋이 성공적으로 되었다면, 로컬의 저장소의 파일을 원격 저장소인 Github로 해당 변경된 내역을 보내주기 위해 푸쉬합니다. `git push origin main` 명령를 사용합니다. 

<img src="/img/@android/jkh_study_fork/study_fork9.png" alt="이미지" style={{ marginBottom: "0px" }} />


다시 한 번 push가 잘 되었는 지 확인하기 위해 로그를 찍어보며 마무리합니다. 

`git log --oneline`

<img src="/img/@android/jkh_study_fork/study_fork10.png" alt="이미지" style={{ marginBottom: "0px" }} />
